# DeepSeek-Coder-V2 调研报告

## 概述

[DeepSeek-Coder-V2](https://github.com/deepseek-ai/DeepSeek-Coder-V2) 是 DeepSeek AI 团队在 2024 年发布的开源代码大语言模型，采用混合专家架构（MoE），专为代码智能和软件开发任务设计。

## 技术参数

| 参数 | 详情 |
|------|------|
| **架构** | Mixture-of-Experts (MoE) |
| **总参数量** | 236B (16B 激活参数) |
| **上下文窗口** | 128K tokens |
| **训练数据** | 87% 代码 + 13% 自然语言（中英文） |
| **支持语言** | 338 种编程语言 |
| **代码能力** | 与 GPT-4 Turbo 持平 |

## 核心能力

1. **代码生成与补全**：多语言代码生成，行级/函数级补全
2. **代码理解**：仓库级上下文理解，跨文件依赖分析
3. **代码修复**：Bug 定位与自动修复
4. **代码重构**：跨文件重构建议

## 部署方式

### 方案一：本地部署（不可行）
- 需要 8x A100/H100 GPU（80GB）才能运行 236B 完整模型
- 量化后仍需 4x A100 级别 GPU
- 成本过高，不适合个人开发环境

### 方案二：API 调用（已有）
- DeepSeek 平台提供云端 API，包含最新 V4 模型
- V4 模型已融合 Coder-V2 全部代码能力并进一步增强
- 本项目已通过 DeepSeek API 集成 V4 模型

## 结论

**无需单独集成 DeepSeek-Coder-V2。**

理由：
1. Coder-V2 是 LLM 模型而非工具/库，无法作为代码库嵌入项目
2. 本地部署需 8 块高端 GPU，成本不可行
3. 本项目已通过 DeepSeek API 使用 V4 系列模型，代码能力已包含并超越 Coder-V2
4. API 方式已是最优解：无需本地 GPU、自动更新到最新模型、按量付费

## 建议

保持现有 DeepSeek API 集成方式。如需增强代码能力：
- 在 AI 配置中切换至 deepseek-v4-pro 或更新型号
- 或添加其他模型提供商（如 Qwen-Coder）作为备选
