# AI IDE 开发文档

## 项目概述

AI IDE 是一个基于 .NET 10 WPF 的 AI 原生集成开发环境，采用 MVVM 架构，集成 DeepSeek 等大语言模型，支持智能体（Agent）模式下的代码生成、文件操作、终端命令执行、联网搜索等全流程 AI 辅助开发。

## 技术架构

### 技术栈
- **运行环境**: .NET 10 WPF (net10.0-windows)
- **语言**: C# 12
- **架构模式**: MVVM (Model-View-ViewModel)
- **代码编辑器**: AvalonEdit (语法高亮、行号显示)
- **数据库**: SQLite (记忆系统)
- **AI 接口**: OpenAI 兼容 API (DeepSeek, Qwen 等)
- **依赖注入**: 手动构造函数注入

## 核心功能

### 1. AI 智能体模式 (Agent Mode)

支持两种对话模式：
- **智能体模式（Agent）**: 完整工具调用能力（文件读写、代码搜索替换、终端执行、联网搜索等）
- **问答模式（QA）**: 只读模式，仅能阅读/搜索项目代码回答问题，不能修改文件

AI 可调用的工具：
- `read_file` / `read_multiple_files` / `search_replace` / `create_file` / `delete_file` - 文件操作
- `move_file` / `copy_file` / `rename_file` - 文件移动/复制/重命名
- `create_dir` / `delete_dir` / `list_dir` - 目录操作
- `search_file` / `grep_code` / `search_codebase` / `search_symbol` - 代码搜索
- `search_web` / `fetch_content` - 联网搜索
- `run_in_terminal` - 终端命令执行
- `scan_project` - 项目结构扫描
- `todo_write` - 任务列表管理
- `build_project` - 项目构建（支持50+语言自动检测）
- `algorithm_*` - 算法库增删改查
- `agent` - 子代理调度（browser / codereview）

### 2. 开发计划模式

- **计划建议内联卡片**: 检测到复杂任务时，在聊天输入区上方显示内联建议卡片（替代弹窗），包含"📋 生成计划"和"⏭ 跳过"两个按钮
- **非阻塞式交互**: 计划建议不会阻塞 UI 线程，用户可随时选择生成计划或跳过直接执行
- **复杂任务检测**: 自动识别复杂开发任务（命中 >=2 个关键词或消息 >300 字符）
- **计划生成**: AI 使用 todo_write 生成详细开发计划，包含目标、技术方案、实施步骤、风险点、验收标准
- **计划详情面板**: 点击"查看详情"按钮可展开/收起 AI 生成的完整计划内容
- **计划操作**: 审阅计划后可"确认并执行"、"重新规划"或放弃
- **自动执行**: 确认计划后 AI 自动按步骤顺序执行，实时更新每步状态（pending → in_progress → completed）
- **进度追踪**: 执行中显示当前步骤名称和进度指示器

### 3. 联网搜索 (Web Search)

- 基于 DuckDuckGo HTML 搜索 API（无需 API Key）
- 支持 AI 主动调用 `search_web` 工具获取实时信息
- 支持 `fetch_content` 获取网页详细内容
- 可配置自动搜索模式：始终询问 / 始终允许 / 始终拒绝
- 搜索结果自动提取标题、摘要和 URL
- 支持时间范围过滤（一天/一周/一月/一年）

### 4. 智能备份 (Smart Backup)

- 在 AI 修改文件前自动备份到 `.aiide-backups/` 目录
- 备份文件带时间戳命名
- 每个文件最多保留 20 个历史备份
- 防止 AI 误改、破坏或丢失代码

### 5. 隐私校验 (Privacy Validation)

- **阻断模式**: 检测身份证号、银行卡号、手机号等个人身份信息，命中后阻断发送
- **提醒模式**: 检测 API Key、Token、密码等凭据信息，弹窗确认后发送
- 使用正则表达式进行模式匹配

### 6. 等待发送队列

- **非阻塞发送**: AI 推理过程中，输入框保持可用，用户可继续输入消息
- **自动排队**: 新消息自动加入等待发送队列，显示在待办列表上方
- **队列 UI**: 展示等待消息数量和内容预览，支持清空队列
- **自动出队**: 当前 AI 回复完成后，自动从队列取出下一条消息发送
- **时间戳**: 每条队列消息显示加入时间

### 7. 停止功能

- 发送/停止合一按钮：对话时显示停止按钮，空闲时显示发送按钮
- 基于 CancellationTokenSource 实现异步流取消
- 可随时停止 AI 对话、推理、代码生成

### 8. 文件引用显示

- 聊天面板显示当前正在操作的引用文件
- 点击文件名可直接在编辑器中打开
- 自动提取完整路径中的文件名显示

### 9. 代码编辑器

- 基于 AvalonEdit 控件
- 语法高亮支持：C#, Python, Java, JavaScript, TypeScript, HTML, CSS, XML/XAML, C/C++, Rust, Go, SQL, JSON, Markdown, YAML, PowerShell, Batch
- 行号显示、代码折叠、上下文菜单（剪切/复制/粘贴）
- 文件标签页：显示当前打开文件，点击切换
- 文件拖放支持

### 10. 文件管理

- 左侧文件树浏览器
- 新建文件/文件夹（带重名检测弹窗）
- 移动文件/目录到新位置（`move_file` 工具，支持跨目录移动）
- 复制文件（`copy_file` 工具，目标存在时自动报错）
- 批量读取多个文件（`read_multiple_files` 工具，最多10个文件）
- 递归列出目录内容（支持关键词过滤和深度控制）
- 项目结构扫描与统计（文件/目录计数、层级展示）
- 文件搜索（grep 正则匹配 + 语义搜索）
- 文件变更追踪列表（修改/新建/删除文件历史）
- 全局搜索替换（跨文件批量搜索和替换，支持预览模式）

### 11. 终端与日志

- 内置终端（cmd.exe），支持命令输入和输出显示
- 日志面板记录所有 AI 调用、工具执行、系统事件
- 底部面板标签化切换

### 12. 记忆系统

- 基于 SQLite 的长期记忆
- 自动记忆项目上下文、开发者习惯
- 设置面板中可查看和管理记忆

### 13. 自维护系统

- 每 10 分钟自动扫描项目健康状态
- 检测常见问题（缺少 .gitignore、代码风格不一致等）
- 支持自动修复和手动处理

### 14. AI 提示词增强

- **文件/目录 CRUD 语义**: 精确理解"查看/创建/修改/删除/移动/复制/重命名"等文件操作意图，自动选择最合适的工具
- **代码 CRUD 响应增强**: search_replace 最佳实践（先读后改、唯一匹配、保持缩进）；create_file 最佳实践（创建父目录、文件头规范）；delete/rename/move 后自动更新引用
- **工具结果增强**: 文件操作后返回行号、文件名等上下文信息，帮助 LLM 做出更精准的后续决策
- **操作后验证**: 修改文件后提示 read_file 验证，可编译项目提示 build_project 确认
- **文件操作语义**: 理解"移动/转移/复制/整理"等自然语言意图，自动选择合适工具
- **路径解析**: 支持相对路径自动解析为绝对路径，从项目根目录推导
- **项目结构感知**: 大规模重构前自动扫描项目结构，理解已有目录组织
- **操作语义映射**: "移动"→move_file, "复制/备份"→copy_file, "同时查看"→read_multiple_files

## API 配置

支持的 AI 提供商：
- DeepSeek (deepseek-v4-pro 等)
- 通义千问 (Qwen系列)
- 自定义 OpenAI 兼容 API

配置项：
- 提供商名称 / API 接口地址
- 模型 ID / 显示名称
- API 密钥
- 模式选择（智能体/问答）
- 最大令牌数

## 构建与运行

```bash
# 还原依赖
dotnet restore

# 编译
dotnet build

# 运行
dotnet run
```

要求：
- .NET 10 SDK
- Windows 10/11 (WPF 依赖)
